# Landscape Process — Plan Before Building

Don't jump into execution. Write a clear plan, validate it is comprehensive, THEN build.

**Why:** User explicitly said "I keep telling you, don't jump into doing things. Remember what has been done in the past." The intelligence v2 pipeline was successful because it was planned, tested on 3 companies, then batch-run. The landscape deserves the same or more rigour — it is "the most important part of our platform."

**How to apply:**
- Before any batch processing, write a detailed plan document
- Validate the plan covers all research sources and quality checks
- Use ALL available search tools (WebSearch, WebFetch from Max subscription) in addition to Jina
- If it takes 3-4 days, that's fine. Quality is non-negotiable.
- Every company must be McKinsey/BCG quality — no shortcuts
- The intelligence v2 improvement came from comprehensive multi-source research. Landscape must match or exceed that.

**Specific gap caught:** I was writing profiles from research briefs that sometimes had zero articles (BofA came back with `confidence: "low"` and no articles). Should have supplemented with WebSearch/WebFetch before writing. The intelligence pipeline always verified sources — landscape must do the same.
