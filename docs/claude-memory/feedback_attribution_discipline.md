Never attribute broad business metrics to specific AI tools unless the source explicitly makes that exact causal claim.

**Why:** JPMorgan incident — Reuters said "gross sales increased 20% with GenAI tools helping teams focus on high-impact work" (vague, broad). Our profile said "20% gross sales growth linked to Connect Coach" (specific, causal). This is fabrication by implication. If someone from Morgan Stanley or BofA reads this, we lose all credibility and cannot scale.

**How to apply:**
- When a source says "AI tools contributed to X% growth" → write exactly that. Do NOT name a specific tool unless the source does.
- When a source says "$X in AI savings" → check if it means AI alone or broader efficiency programmes (AI + automation + ops). Qualify accordingly.
- When a source says "$X AUM in accounts using [tool]" → that's portfolio size, NOT growth attributable to the tool.
- The structure is: [metric] + [what the source actually said about AI's role] — never tighten the attribution beyond what the source claims.
- Before publishing any capability evidence that links a metric to a tool, ask: "Did the source say this tool caused this number?" If no → soften the language.

**Pattern to watch for:** AI tool metrics and business metrics appearing in the same sentence without explicit causal language from the source. Proximity implies causation even when a disclaimer follows.

**Companies fixed (2026-04-05):** JPMorgan (20% → GenAI broadly cited), Lloyds (£0.8B → broader efficiency programme), Santander (€200M → efficiency gains incl. AI), BofA ($211B → AUM in accounts using Erica, not growth from Erica).

**Also fixed:** Smart Monitor removed from JPMorgan wealth management profile entirely — it's an Asset Management tool (analysts/PMs), out of scope per the profile's own scope_note.
